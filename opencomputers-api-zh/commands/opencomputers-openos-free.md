# free

## 概述

输出当前 Lua 运行环境可见的总内存、已用内存和空闲内存估算值。

## 可用性

- 仓库：`opencomputers`
- 系统：`openos`
- 类型：`command`

## 语法

```lua
free
```

## 用途

`free` 会多次采样 `computer.freeMemory()`，并在采样过程中让出调度机会，以便垃圾回收器有机会运行。随后它会根据观测到的最高空闲内存值，输出三项统计：

- `Total`：当前计算机分配给 Lua 的总可用内存。
- `Used`：估算出的已使用内存。
- `Free`：估算出的可用空闲内存。

## 输出说明

该命令显示的是 Lua 层面的内存情况，不是宿主操作系统意义上的物理内存，也不是整台 Minecraft 服务器的内存占用。

数值含义如下：

- `Total` 约等于 `computer.totalMemory()`。
- `Free` 基于多次采样后得到的最高空闲值。
- `Used` 由总内存减去空闲内存计算得出。

## 使用示例

```lua
free
```

示例输出通常类似：

```text
Total: 1048576
Used:   262144
Free:   786432
```

## 适用场景

- 判断脚本是否接近内存上限。
- 对比垃圾回收前后的内存占用变化。
- 在加载大型表、解析文件或运行复杂算法前后进行快速检查。

## 备注

- `Used` 和 `Free` 是经过多次采样后的近似结果，不保证是某一个瞬时绝对值。
- 如果脚本刚刚释放了大量对象，命令往往会比单次读取 `computer.freeMemory()` 更接近真实可回收后的状态。
